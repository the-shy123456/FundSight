# 系统架构蓝图

## 1. 产品定位

基金智能分析与决策辅助系统。

边界：

- 输出风险约束下的建议，不承诺最大收益
- 输出证据链、置信度、适用条件
- 后续如果提供个性化投顾能力，必须单独接入合规流程

## 2. 分层设计

```text
客户端层
├── Web
├── Windows Desktop
└── Mobile

接入层
├── API Gateway / BFF
└── 鉴权、限流、缓存

能力层
├── 基金画像服务
├── 分析评分服务
├── 组合建议服务
├── 深度搜索服务
├── 文章解析服务
└── 回测与模拟交易服务

数据层
├── PostgreSQL
├── ClickHouse
├── Redis
├── Object Storage
└── Vector Store
```

## 3. 语言与技术路线

### 第一阶段

- Web：静态原型，后续平滑迁移到 Next.js
- 分析服务：Python
- 数据：样例数据

### 第二阶段

- Web：Next.js
- Windows：Tauri 2
- Mobile：React Native / Expo
- API Gateway：Go
- 分析与研究：Python
- 性能热点：Rust

## 4. 核心链路

### 基金分析

1. 获取基金基础资料、净值序列、费率、经理信息
2. 计算收益、波动、回撤、动量、风格稳定性
3. 输出基金诊断标签与风险说明

### 组合建议

1. 收集用户风险等级、预算、投资周期
2. 按风险偏好筛选基金池
3. 结合质量评分、动量、费用输出候选组合
4. 给出金额分配与操作节奏

### 深度研究

1. 抓取公告、季报、年报、招募说明书、新闻
2. 抽取事件、观点、风险、经理变化
3. 关联基金与主题
4. 生成可解释结论

## 5. 性能设计

- 热门基金分析结果预计算
- API 响应以缓存优先
- 长耗时任务异步化
- 客户端首屏只取关键指标，深度分析按需加载
- 客户端缓存最近浏览、自选和最近分析结果
